# WSTS
Web service Trouble Shooting

1. django를 이용해 로컬 웹서버에서 500000 이하의 소수 counting을 진행하고 runtime을 측정합니다.
2. 2개의 탭에서 연달아 실행했을 때, 두 번째로 실행한 탭은 첫 번째 실행한 탭의 counting이 끝날때까지 block되어 실행시간이 길어집니다.
3. 해당 현상이 발생하는 이유는 django에서 개설한 웹서버가 싱글 프로세스로 실행되어 작업이 끝날 때까지 다른 액세스를 처리할 수 없기 때문입니다.
4. waitress wsgi를 활용해 병렬 스레드를 활용해 하나의 작업을 수행하는 동안 다른 스레드를 활용해 추가 액세스를 처리할 수 있습니다.
